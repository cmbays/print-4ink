# Web Research Checklist

**Vertical**: [VERTICAL NAME]
**Competitor**: [COMPETITOR NAME]
**Date**: [DATE]

---

## Company & Product

- [ ] Company name, website URL
- [ ] Founder/team: Who built it? Background? Team size?
- [ ] User base: How many users? What types of businesses?
- [ ] Pricing: Tiers, free trial, per-user vs flat rate?
- [ ] Positioning: How do they describe themselves?

## Technology

- [ ] Frontend framework (view-source, JS bundle names, CDN URLs)
- [ ] API integrations (what services do they connect to?)
- [ ] Data sources (vendor catalogs, stock levels, pricing APIs)
- [ ] Mobile support (responsive? native app?)

## Feature Set for [VERTICAL]

- [ ] List all features related to this vertical (from marketing pages, help docs)
- [ ] Check feature comparison pages vs competitors
- [ ] Note features marked as "coming soon" or "beta"
- [ ] Check changelog/release notes for recent additions

## Recent Updates

- [ ] Blog posts from last 12 months about this vertical
- [ ] Release notes mentioning this vertical's features
- [ ] Bug fixes related to this vertical (indicates pain points)
- [ ] Roadmap items if publicly shared

## Community & Reviews

- [ ] User reviews mentioning this vertical (G2, Capterra, Reddit)
- [ ] Social media complaints or praise
- [ ] Forum posts about workarounds (indicate missing features)
- [ ] Support documentation gaps

## Competitive Landscape

- [ ] Who else serves this vertical? (3-5 competitors)
- [ ] How do they compare on pricing?
- [ ] What's each one's main differentiator?
- [ ] Market gaps: What does nobody do well?
