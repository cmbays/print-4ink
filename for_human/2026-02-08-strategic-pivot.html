üîç The Methodology: 4-Phase Approach (per Vertical)

Each vertical follows this repeatable cycle:

Phase 1: Discovery (2-3 days per vertical)

Deeply understand Print Life's current experience and 4Ink's pain points

  • Print Life Analysis: Screenshot review + trial walkthrough, extract feature list, UI patterns, workflow steps
  • User Interview: 30-45 min with Chris, cover current workflow ‚Üí pain points ‚Üí interconnections ‚Üí wishlist
  • Journey Mapping: Document current workflow steps, identify friction points, rate severity

Phase 2: Scope Definition (1 day per vertical)

Define what we'll build (CORE), what we'll mock (PERIPHERAL), what we'll minimize (INTERCONNECTIONS)

  • CORE: Must-have workflows that demonstrate superior UX
  • PERIPHERAL: Nice-to-haves shown in UI but non-functional (e.g., "PDF download" button that shows "Coming in Phase 2")
  • INTERCONNECTIONS: Minimal representations for user journey continuity (e.g., customer dropdown links to Customer Management vertical but doesn't build full CRUD)

Phase 3: Build Execution (2-4 days per vertical)

Implement mockup with design system adherence and quality gates

  • Component Planning: Identify reusable components, build shared pieces first
  • Screen Implementation: Use frontend-builder agent with screen-builder skill, follow design system
  • Quality Gate: Run quality-gate skill audit, fix Critical/High issues

Phase 4: Demo & Iteration (1 day per vertical)

Get feedback, refine before moving to next vertical

  • Live Demo: Screen share, compare side-by-side to Print Life, ask feedback questions
  • Rate Dimensions: User rates Clarity, Speed, Polish, Value (target 8+ average)
  • Iterate: If needed, implement changes and re-run quality gate (max 1 day)

‚ö° Quoting Vertical: Detailed Pilot Plan

Quoting is the pilot vertical that establishes the repeatable 4-phase methodology.

Why Quoting?

  • Most customer-facing (sets first impression)
  • Clearest feedback from 4Ink (they showed us this vertical specifically)
  • Establishes patterns for form-heavy workflows (applies to Invoicing, Pricing Matrix)
  • Defines interconnection patterns (customer selection, pricing, reporting)

What We're Building (CORE)

  • Quotes List (/quotes) ‚Äî Browse quotes, filter by status, search by customer/number
  • Quote Detail (/quotes/[id]) ‚Äî View full quote breakdown, totals, action buttons
  • New Quote Form (/quotes/new) ‚Äî Dynamic line items, real-time pricing, inline editing (no modals!)

What We're Showing But Not Building (PERIPHERAL)

  • Customer creation modal (simple name+email only)
  • Artwork upload area (placeholder, doesn't process files)
  • PDF download button (shows "Coming in Phase 2")
  • "Send to Customer" email preview (mockup modal, doesn't send)

What We're Minimizing (INTERCONNECTIONS)

  • Customer Selection: Simple searchable dropdown (hints at full Customer Management vertical)
  • Pricing Formula: Hard-coded calculation (hints at Pricing Matrix vertical)
  • "Convert to Invoice" Button: Non-functional, shows message (hints at Invoicing vertical)
  • Quote Totals: Captured but not displayed (will feed into Reporting vertical)
